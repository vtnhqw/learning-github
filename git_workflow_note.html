Git + GitHub deploy workflow reference
GitHub repo
→
clone to PC
→
edit in VS Code
→
git push
→
live site ✓
Steps
1

Clone — do this once only

git clone https://github.com/your-username/your-blog-repo.git
2

Open in VS Code

cd your-blog-repo
code .

Or drag the folder into VS Code manually.

3

Edit your files

style.css for UX · index.html for layout · save with Ctrl+S

4

Push to GitHub

Open terminal in VS Code with Ctrl + `

git add .
git commit -m "updated blog header style"
git push origin main
5

Auto-deploy fires ~1–2 min

GitHub Actions triggers automatically. Watch it: repo → Actions tab.

Every editing session
git pull origin main     # get latest before editing

# ... make your changes ...

git add .
git commit -m "what you changed"
git push origin main
VS Code shortcut — Use the Source Control panel (branch icon, left sidebar) to stage, commit, and push with clicks instead of terminal commands.
Next step — Set up deploy.yml in your repo so auto-deploy is ready to go.
